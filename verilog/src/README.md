# verilog/src

このディレクトリには、5段パイプラインプロセッサ（`5plp_1`）の設計データ（RTL）が格納されています。
論理合成の対象となるのは、ここで設計したプロセッサのみです。

## 📁 ディレクトリ内の主要ファイル
* **`5plp_1.v`**（または関連する設計ファイル）: 今回開発した5段パイプラインプロセッサのコアモジュールです。


*※注意: テストベンチやシミュレーション関連のファイル（`simulate.sh` など）は `../tb/` ディレクトリを使用しますが、本リポジトリには研究室共通の資産（キャッシュ関連など）は含めていません。*

---

## 🚀 実行手順（シミュレーション 〜 論理合成）

本ディレクトリのコードをテストし、論理合成を行うための手順です。

### 1. 準備（研究室共通ファイルの配置）
シミュレーションと論理合成を行うために、CPCラボで配布されている以下のファイルを noVNC 環境の指定ディレクトリにアップロードしてください。

* `cash.sh`, `top.sh` など（配布されたスクリプト） ➡️ `/verilog/src/` へアップロード
* ※ 必要なテストベンチファイル ➡️ `/verilog/tb/` へアップロード 

### 2. シミュレーション（動作確認）
`../tb/` ディレクトリに移動し、シミュレーションを実行してプロセッサが正しく動作するか確認します。

```bash
cd ../tb/
# シミュレーションの実行（例）
sh ./simulate.sh 5plp_1
```

## 3. 論理合成（Yosys）

シミュレーションで正常動作が確認できたら、`/verilog/src/` に戻り、自身のプロセッサ（**5plp_1**）を対象に論理合成を行います。

> [!IMPORTANT]
> **注意：** 演習で配布されているキャッシュモジュールは論理合成を想定していない（非現実的な時間がかかる）ため、**合成対象には含めないでください。**

---

### 実行手順

ターミナルで以下のコマンドを実行し、論理合成を開始します。

```bash
cd ../src/
sh ./yosys_stats.sh 5plp_1
```

## 💡 後輩へのアドバイス（論理合成の壁を越えるために）

シミュレーションではパスしても、Yosysでの論理合成やその後のLibreLane（配置配線）では、VSCodeなどの構文チェックでは出なかったエラーや警告が大量に出るはずです。ここで時間を取られることを覚悟しておいてください。

### ⚠️ よくある躓きポイント

* **ラッチの意図しない生成**
    * `always` ブロックでの条件分岐（`if`, `case`）が網羅されていない（`else` や `default` がない）と、合成ツールが値を保持しようとして**ラッチ**を生成します。これはタイミングエラーの温床になります。
* **非現実的な記述**
    * シミュレーション上は動いても、実際のハードウェア（回路）としてマッピングできない記述はエラーになります。
    * 例：複雑な初期化、遅延指定（`#1` など）、ループ境界が不明瞭な `for` 文など。
* **キャッシュモジュールは合成しない**
    * 前述の通り、演習用のキャッシュは合成できません。**コア部分（5plp_1）のみ**にフォーカスして合成を進めてください。

> [!TIP]
> 「シミュレーションはゴールではなく、ようやく合成のスタートラインに立った」という意識で取り組むのがコツです！
